# Distiller（万物可蒸馏）— 执行计划

> 版本：v1.0 | 日期：2026-05-30

---

## 开发原则

1. **逐阶段推进**：每个阶段完成后验证，确认无误再进入下一阶段
2. **可运行交付**：每个阶段的产出必须是可运行、可验证的
3. **不跳跃依赖**：严格按顺序执行，不做跨阶段的功能
4. **代码简洁**：不引入未使用的抽象，不过度设计
5. **先确认再动手**：不确定的地方先问，不猜测
6. **每阶段验证**：运行 `python main.py` 确认功能正常

## 阶段划分

### 阶段 1：项目骨架
- 创建完整目录结构
- 编写 main.py（最小 Eel 应用，能打开窗口）
- 编写 index.html（最小页面，显示标题）
- 编写 requirements.txt
- **验证标准**：运行 `python main.py` 能打开桌面窗口，显示 Distiller 标题

### 阶段 2：配置 & API 连接
- 实现 backend/config.py（配置读写）
- 实现 backend/database.py（SQLite 建表）
- 实现 backend/llm_client.py（DeepSeek API 调用）
- 前端设置页面（输入 API Key、测试连接）
- **验证标准**：输入 API Key 后能发送消息并收到 DeepSeek 回复

### 阶段 3：知识库核心
- 实现 backend/embedding.py（本地嵌入模型加载）
- 实现 backend/knowledge_base.py（ChromaDB 管理）
- 实现"粘贴文本录入"功能
- 实现基础 RAG 问答（检索 + 生成）
- **验证标准**：粘贴一段文本后，能基于该内容回答问题

### 阶段 4：文档解析 & 文件上传
- 实现 backend/document_parser.py（txt/docx/pdf）
- 前端文件拖拽上传区域
- 文件上传进度提示
- **验证标准**：上传 docx/pdf 文件后内容自动录入知识库并可检索

### 阶段 5：蒸馏引擎
- 实现 backend/distillation.py（Prompt Chain）
- 三种蒸馏模式：人物方法论 / 领域知识 / 风格特征
- 蒸馏结果的结构化输出（Markdown 报告）
- **验证标准**：录入人物资料后运行蒸馏，生成结构化的方法论报告

### 阶段 6：聊天界面打磨
- 完整聊天 UI（消息气泡、Markdown 渲染、代码高亮）
- 流式输出（逐 token 显示）
- 对话历史管理
- 知识库切换
- **验证标准**：完整的多轮对话体验，流式输出流畅

### 阶段 7：全网搜索
- 实现 backend/web_search.py（DuckDuckGo）
- 前端搜索界面（输入关键词 → 搜索结果预览 → 确认录入）
- **验证标准**：搜索人名后自动抓取内容并录入知识库

### 阶段 8：Skills 系统
- 实现 backend/skill_manager.py
- 内置 3 个默认模板 + 2 个默认角色
- 前端 Skills 管理界面（新建/编辑/删除）
- Skills 注入对话流程
- **验证标准**：选择不同 Skill 后 AI 回答风格/视角明显变化

### 阶段 9：风格生成 & 打磨
- 风格蒸馏 + 生成功能
- 错误处理、空状态、加载态
- 设置完善（嵌入模型选择、检索参数等）
- **验证标准**：录入方文山歌词后能生成模仿风格的新歌词

### 阶段 10：打包发布
- PyInstaller 配置
- 打包为 Distiller.exe
- 在干净环境测试
- **验证标准**：独立 .exe 运行正常
